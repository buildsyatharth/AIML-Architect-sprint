# AIML Architect Sprint: 300 Days to Mastery

## The Vision
This repository is a daily log of my journey to becoming an AI/ML Architect. 300 days of consistent coding, mathematical foundations, and building real-world intelligent systems.

## Phase 1: Python Fundamentals & Logic

### Day 1: The Foundation
* **Focus:** Environment Setup & Basic Syntax
* **Key Tasks:**
    * Configured VS Code and Git workflow.
    * Mastered Variables & Data Types (`int`, `float`, `str`, `bool`).
    * Implemented Arithmetic Operators and Type Casting.
    * **Project:** Built a basic calculator to compute sums and area dimensions.
* **Learning:** Coding is 50% logic and 50% troubleshooting.

### Day 2: The Decision Maker
* **Focus:** Strings & Conditional Logic
* **Key Tasks:**
    * **String Manipulation:** Slicing, indexing (positive/negative), and escape sequences.
    * **String Methods:** Used `.find()`, `.replace()`, `.count()`, and `.capitalize()`.
    * **Control Flow:** Mastered `if`, `elif`, and `else` structures.
* **Practice Problems (9 Solved):**
    * Traffic Light Simulator, Voting Eligibility, Grade Calculator, Even/Odd Checker, and Divisibility tests.
    * **Hardest Challenge:** Finding the greatest of 4 numbers using nested `if` statements (Pure logic, no `max()` function).

### Day 3: Recovery & Strategic Planning
* **Focus:** Weekly Roadmap & Sustainability
* **Status:** Dedicated rest day to recover from heat/dehydration.
* **Strategic Goal:** Planning the transition from simple scripts to Data Structures.
* **Why Rest?** An Architect’s most valuable tool is a clear mind. Burnout is the enemy of a 300-day sprint.
